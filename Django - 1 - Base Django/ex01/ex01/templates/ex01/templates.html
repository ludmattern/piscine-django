Ex01 : Moteur de template.

Le moteur de template Django

Le moteur de template par défaut de Django (Django Template Language - DTL) permet de séparer la logique de présentation du code Python. Il utilise une syntaxe simple avec des balises et des filtres pour générer du HTML dynamique.

Les Blocs

Les blocs permettent de définir des zones dans un template de base qui peuvent être surchargées par les templates enfants. C'est le cœur du système d'héritage de templates. 

{% templatetag openblock %} block nom_du_bloc {% templatetag closeblock %}
    Contenu par défaut
{% templatetag openblock %} endblock {% templatetag closeblock %}

Un template enfant peut ensuite remplacer ce contenu en redéfinissant le bloc.

Les boucles for ... in

La balise for permet d'itérer sur une séquence (liste, queryset, etc.). 

{% templatetag openblock %} for item in liste {% templatetag closeblock %}
    {{ item }}
{% templatetag openblock %} endfor {% templatetag closeblock %}

Exemple avec les données passées :

    {% for item in exemple_liste %}
  • {{ item }}
    • {% endfor %}

Les structures de contrôle if

La balise if permet d'afficher conditionnellement du contenu. 

{% templatetag openblock %} if condition {% templatetag closeblock %}
    Affiché si la condition est vraie
{% templatetag openblock %} elif autre_condition {% templatetag closeblock %}
    Affiché si autre_condition est vraie
{% templatetag openblock %} else {% templatetag closeblock %}
    Affiché sinon
{% templatetag openblock %} endif {% templatetag closeblock %}

Exemple :

{% if afficher_message %} Le message est affiché car la variable afficher_message est True. {% else %} Ce texte ne s'affichera pas. {% endif %}

L'affichage de variables passées par contexte

Les variables sont passées depuis la vue via le contexte et affichées avec la syntaxe {% templatetag openvariable %} nom_variable {% templatetag closevariable %}. 

{{ ma_variable }}

Exemple : La variable exemple_variable vaut : {{ exemple_variable }}
