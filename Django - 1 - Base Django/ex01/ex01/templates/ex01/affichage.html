Ex01 : Processus d'affichage d'une page statique.

Le processus de la requête à la réponse

  1. 1. La requête HTTP

    L'utilisateur entre une URL dans son navigateur (ex: /ex01/affichage). Le navigateur envoie une requête HTTP au serveur Django.

  2. 2. Le routage URL (urls.py)

    Django reçoit la requête et consulte le fichier urls.py pour trouver le pattern d'URL correspondant. Chaque URL est associée à une vue spécifique. 

    path('affichage', views.affichage, name='affichage')

  3. 3. La vue (views.py)

    La vue correspondante est appelée. Elle peut effectuer des traitements (accès base de données, calculs, etc.) et prépare le contexte à passer au template. 

    def affichage(request):
    return render(request, 'ex01/affichage.html')

  4. 4. Le template

    Django utilise le moteur de template pour rendre le fichier HTML. Le template peut contenir des variables, des boucles et des conditions qui sont interprétées par le moteur.

  5. 5. La réponse HTTP

    Une fois le template rendu en HTML pur, Django renvoie une réponse HTTP au navigateur avec le contenu HTML généré. Le navigateur affiche alors la page.

Schéma récapitulatif

Navigateur → Requête HTTP → urls.py → views.py → template.html → Réponse HTTP → Navigateur
